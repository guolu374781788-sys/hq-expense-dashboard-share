# 费用看板部署说明

当前目录已经是可直接部署的静态站点，无需构建。

## 目录说明

- `index.html`：站点入口
- `app.js`：页面交互与数据
- `styles.css`：样式文件
- `.nojekyll`：部署到 GitHub Pages 时，避免被 Jekyll 处理

## 方案 1：GitHub Pages

适合：
- 需要一个固定公开链接
- 代码也希望顺手备份到 GitHub

操作步骤：

1. 新建一个 GitHub 仓库，比如 `expense-dashboard`
2. 把当前目录里的文件上传到仓库根目录
3. 在 GitHub 仓库里打开 `Settings`
4. 找到 `Pages`
5. 在 `Build and deployment` 里选择：
   - `Source`: `Deploy from a branch`
   - `Branch`: `main` / `/root`
6. 保存后等待 1-3 分钟

产出的链接通常是：

`https://<你的GitHub用户名>.github.io/<仓库名>/`

例如：

`https://your-name.github.io/expense-dashboard/`

## 方案 2：Vercel

适合：
- 想最快拿到一个可分享链接
- 不想自己配服务器

操作步骤：

1. 在这份目录打开终端
2. 首次安装 Vercel CLI：

```bash
npm i -g vercel
```

3. 登录：

```bash
vercel login
```

4. 在当前目录执行：

```bash
vercel
```

5. 按提示完成首次部署

部署成功后，终端会直接输出一个可访问链接。

如果要发正式生产链接，再执行：

```bash
vercel --prod
```

## 分享给其他人

拿到链接后，直接把这个 URL 发给对方即可。

如果是 GitHub Pages：
- 链接固定，适合长期分享

如果是 Vercel：
- `--prod` 生成的链接更适合正式分享

## 更新内容后如何重新发布

### GitHub Pages

把新文件重新提交并 push 到仓库，Pages 会自动更新。

### Vercel

在当前目录重新执行：

```bash
vercel --prod
```

## 推荐

如果你只是想尽快分享，优先用 `Vercel`。

如果你想保留一个长期稳定、可持续更新的地址，优先用 `GitHub Pages`。
